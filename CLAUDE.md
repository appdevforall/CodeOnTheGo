### Known Issue: DiagnosticsFormatter Shows 0%

`DiagnosticsFormatter` in `:app` always reports 0% due to a JaCoCo fingerprint mismatch — not an actual coverage gap.
Its inner lambdas report 100%, confirming the 19 DiagnosticsFormatterTest tests do execute.

**Root cause:** The `app` module uses AGP offline JaCoCo instrumentation (`transformV8DebugClassesWithAsm`).
KAPT stub generation runs between production compilation and the ASM transform, triggering incremental
recompilation of some app classes. The exec file records the CRC of the pre-recompile bytecode; the
report reads the post-recompile class on disk → CRC mismatch → 0%.

Library modules (`:git-core`, `:common`, `:lsp:indexing`) are unaffected.
